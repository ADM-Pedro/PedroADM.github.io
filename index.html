Crie um site completo, bonito, moderno e responsivo para celular, com visual premium e estilo de plataforma/app, para um projeto chamado “Diário do DK”. IMPORTANTE: - Gere TUDO em um único arquivo `index.html` - Inclua HTML, CSS e JavaScript no mesmo arquivo - Não use React - Não use frameworks - Não use bibliotecas externas além do Google Fonts - O código deve estar limpo, funcional e pronto para subir no GitHub Pages - O site deve funcionar sozinho ao abrir o arquivo - Não explique o código, apenas entregue o código completo OBJETIVO DO SITE: O site é para jogadores de Brawl Stars se cadastrarem para participar de partidas pagas no “Diário do DK”. ESTILO VISUAL: - Visual moderno, bonito, limpo e profissional - Inspirado em dashboards e landing pages premium - Fundo escuro com gradiente elegante - Cores principais: azul escuro, roxo, preto, com detalhes em verde e amarelo - Aparência de produto real, não de página simples - Bordas arredondadas - Sombras suaves - Bom espaçamento - Layout centralizado - Design mobile first - Deve ficar muito bonito no celular TIPOGRAFIA: - Usar Google Fonts - Título principal com fonte forte, como Anton - Restante do texto com fonte como Poppins ESTRUTURA DO SITE: 1. HERO / TOPO Criar uma seção principal bem bonita contendo: - Título grande: “Diário do DK” - Subtítulo: “Partidas de Brawl Stars valendo dinheiro real” - Um pequeno selo/status visual tipo: - “Vagas limitadas” ou - “Online” - Abaixo do título, criar 3 cards pequenos e bonitos lado a lado mostrando: - Entrada: R$5 - Por kill: +R$3 - Por vitória: +R$10 2. PAINEL DE VAGAS Criar uma seção moderna mostrando o status da próxima sala: - Vagas totais: 9 - Inscritos: valor dinâmico via JavaScript - Restantes: valor dinâmico via JavaScript Essas 3 informações devem ficar em 3 cards bonitos lado a lado. Abaixo disso, mostrar: - Uma barra de progresso horizontal bonita - Texto centralizado como: “7/9 vagas preenchidas” Se as vagas restantes forem 0, mostrar uma mensagem de alerta elegante: - “Lotado — todas as vagas foram preenchidas” Se ainda houver vagas, mostrar: - “Garanta sua vaga antes que acabe” Criar a lógica em JavaScript com variáveis fáceis de editar, por exemplo: - totalVagas = 9 - inscritos = 7 3. FORMULÁRIO Criar uma seção bonita com título como: - “Entrar” ou - “Garantir minha vaga” O formulário deve ter estes campos: - Nome completo - Telefone - Nick do Brawl Stars O design do formulário deve ser moderno: - Inputs grandes - Fundo escuro - Bordas suaves - Espaçamento bonito - Totalmente responsivo Botão principal: - Grande - Chamativo - Bonito - Texto: “Garantir minha vaga” 4. MENSAGEM APÓS ENVIO Ao enviar o formulário: - Não precisa enviar para backend - Não precisa salvar em banco - Apenas impedir o reload da página com JavaScript - Esconder o formulário - Mostrar uma mensagem bonita de confirmação A mensagem deve dizer exatamente: “✅ Cadastro enviado com sucesso! Agora faça o Pix de R$5,00 para garantir sua vaga. Chave Pix: SUA_CHAVE_AQUI Depois do pagamento, envie o comprovante com: - Seu nome - Seu nick do Brawl Stars Exemplo: Pedro - DKShadow ⚠️ A vaga só é confirmada após o pagamento.” Essa mensagem deve aparecer em um card bonito, destacado e elegante. 5. RANKING Criar uma seção chamada: - “Ranking dos players mais ativos” Mostrar Top 5 jogadores com layout bonito. Cada item deve mostrar: - posição - nome - estatística Exemplo: 1. DKShadow — 31 kills 2. Brabo777 — 24 kills 3. LoboBS — 20 kills 4. NinjaPro — 18 kills 5. MitoBR — 15 kills Regras do design: - destacar o top 1 - usar medalhas ou estilo visual elegante - não exagerar - manter limpo e moderno 6. DETALHES DE UX - Adicionar hover suave nos botões - Adicionar transições leves - A barra de progresso deve animar suavemente - Tudo deve parecer profissional - Não deixar visual infantil - Não deixar poluído - Não encher de texto desnecessário 7. RESPONSIVIDADE - O site deve ficar excelente em celular - Também deve funcionar bem no desktop - O layout deve se ajustar automaticamente 8.
